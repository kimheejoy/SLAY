# 디자인 가이드라인 — SLAY

> **버전** 1.0 · **작성일** 2026-03-19
> 시니어 사용자를 위한 디지털기기 시뮬레이션 학습 서비스 디자인 원칙

---

## 1. 디자인 원칙

SLAY의 모든 화면은 다음 4가지 원칙을 기반으로 설계한다.

| 원칙 | 설명 |
|------|------|
| **단순함** | 화면당 주요 행동 1–2개, 불필요한 정보 제거 |
| **명확함** | 아이콘은 항상 텍스트 레이블과 함께 제공 |
| **안전함** | 실수해도 되돌릴 수 있는 구조, 부정적 언어 금지 |
| **일관성** | 동일한 색상·간격·컴포넌트를 모든 화면에서 반복 사용 |

## 2. 색상 시스템

### 2.1 CSS 변수 정의

```css
:root {
  --color-primary:           #122961;  /* 진한 네이비 */
  --color-primary-dark:      #0A1A42;  /* 네이비 호버/활성 */
  --color-secondary:         #FCE300;  /* 옐로우 */
  --color-secondary-dark:    #D4BD00;  /* 옐로우 호버/활성 */
  --color-bg:                #F5F5F5;  /* 페이지 배경 */
  --color-surface:           #FFFFFF;  /* 카드·모달 배경 */
  --color-text-primary:      #111111;  /* 기본 텍스트 */
  --color-text-on-primary:   #FFFFFF;  /* Primary 배경 위 텍스트 */
  --color-text-on-secondary: #111111;  /* Secondary 배경 위 텍스트 */
  --color-border:            #CCCCCC;  /* 구분선·테두리 */
  --color-success:           #1A6B2A;  /* 완료·성공 상태 */
  --color-hint:              #122961;  /* 힌트 말풍선 */
  --min-touch:               48px;     /* 최소 터치 영역 */
  --gap:                     20px;     /* 기본 간격 */
}
```

### 2.2 색상 용도

| 토큰 | 색상 | 용도 |
|------|------|------|
| `--color-primary` | `#122961` | 헤더 배경, 주요 버튼, 탭 활성 상태 |
| `--color-secondary` | `#FCE300` | CTA 버튼, 튜토리얼 힌트 하이라이트 |
| `--color-bg` | `#F5F5F5` | 전체 페이지 배경 |
| `--color-surface` | `#FFFFFF` | 메뉴 카드, 모달, 장바구니 항목 |
| `--color-success` | `#1A6B2A` | 결제 완료, 담기 확인 등 성공 상태 |

### 2.3 명도 대비 기준 (WCAG 2.1 AA)

| 텍스트 유형 | 최소 대비율 | 예시 조합 |
|------------|------------|----------|
| 일반 텍스트 (18px 미만) | 4.5:1 | `#FFFFFF` on `#122961` |
| 대형 텍스트 (30px 이상 또는 Bold 24px 이상) | 3:1 | `#111111` on `#FCE300` |

## 3. 타이포그래피

### 3.1 폰트 패밀리

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
             Helvetica, Arial, sans-serif;
```

시스템 폰트를 사용하여 별도 로딩 없이 기기 최적화 렌더링을 제공한다.

### 3.2 폰트 크기 체계

| 용도 | 크기 | 굵기 |
|------|------|------|
| 기본 본문 | 18px | 400 |
| 소제목 / 버튼 | 20px | 600 |
| 제목 / 헤더 | 24px 이상 | 700 |

- 최대 폰트 굵기: **700** (800 이상 사용 금지)
- 시니어 가독성을 위해 본문 최소 18px 유지

## 4. 간격 및 레이아웃

### 4.1 기본 규칙

| 항목 | 값 |
|------|-----|
| 기본 간격 (`--gap`) | 20px |
| 요소 간 최소 간격 | 20px |
| 화면 좌우 패딩 | 20px |
| 최소 터치 영역 | 48×48px |

### 4.2 해상도

- 기준 해상도: **800×1280** (세로 태블릿)
- 반응형 지원 범위: 320px – 1280px
- 주 사용 환경: 태블릿 또는 스마트폰 세로 모드

### 4.3 레이아웃 원칙

- 화면당 주요 액션은 **1–2개**로 제한
- 중요한 버튼은 화면 하단 고정 배치
- 뒤로가기/취소 버튼은 **모든 화면 좌상단**에 고정, 레이블 필수 포함

## 5. 컴포넌트 가이드

### 5.1 버튼

| 유형 | 배경색 | 텍스트색 | 용도 |
|------|--------|---------|------|
| Primary | `--color-primary` | `--color-text-on-primary` | 일반 주요 행동 |
| CTA (Call to Action) | `--color-secondary` | `--color-text-on-secondary` | 결제, 담기 등 핵심 행동 |
| Ghost / 취소 | 투명 + 테두리 | `--color-primary` | 취소, 뒤로가기 |

- 모든 버튼의 터치 영역: 최소 **48×48px**
- 버튼 텍스트: 최소 18px, 아이콘 병기 시 텍스트 레이블 필수

### 5.2 카드

- 배경: `--color-surface` (`#FFFFFF`)
- 테두리: 없음 (그림자로 구분)
- 내부 패딩: 16–20px
- 모서리 반경: 12px

### 5.3 힌트 말풍선 (튜토리얼 전용)

- 배경: `--color-secondary` (`#FCE300`)
- 텍스트: `--color-text-on-secondary` (`#111111`)
- 대상 요소 위 또는 아래 자동 배치 (화면 밖 이탈 방지)
- 하이라이트 영역 외 배경: 반투명 어두운 오버레이

### 5.4 아이콘

- 항상 텍스트 레이블과 함께 사용 (단독 사용 금지)
- 최소 크기: 24×24px (터치 영역은 48×48px 확보)

## 6. UX 문구 가이드

### 6.1 부정적 언어 금지

시니어 사용자의 심리적 부담을 줄이기 위해 부정적 표현을 사용하지 않는다.

| 사용 금지 | 대체 표현 |
|----------|----------|
| 오류가 발생했습니다 | 다시 해볼까요? |
| 실패했습니다 | 확인이 필요해요 |
| 잘못된 입력입니다 | 이렇게 해보세요 |
| 찾을 수 없습니다 | 다른 방법으로 찾아볼게요 |

### 6.2 행동 유도 문구

- 튜토리얼 힌트: `"여기를 눌러보세요 👆"`
- 완료 메시지: 성취감을 주는 긍정적 표현 사용
- 버튼 레이블: 동사형으로 명확하게 (`주문하기`, `담기`, `결제하기`)
