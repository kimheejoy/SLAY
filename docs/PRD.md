# PRD — SLAY: 시니어 디지털기기 시뮬레이션 학습 서비스

> **버전** 1.0 · **작성일** 2026-03-19

---

## 1. 서비스 개요

### 1.1 서비스명
**SLAY** (Senior Learning As You go)

### 1.2 한 줄 정의
시니어가 실제 기기를 쓰기 전, 실수 없이 안전하게 연습할 수 있는 디지털기기 시뮬레이션 학습 서비스

### 1.3 핵심 가치
| 가치 | 설명 |
|------|------|
| **안전한 연습** | 실수해도 아무 일도 일어나지 않는 가상 환경 |
| **즉각적 피드백** | 무엇을 눌러야 하는지 바로 알 수 있는 힌트 |
| **자립 학습** | 도움 없이 스스로 해냈다는 성취감 제공 |
| **실생활 직결** | 실제 카페, 교통, 은행 기기와 동일한 UI 패턴 |

---

## 2. 문제 정의 및 목표

### 2.1 해결하려는 문제
- 시니어는 키오스크·무인기기 앞에서 **줄이 밀릴까 봐** 눌러보지 못하고 포기한다.
- 가족·직원에게 물어보기 **민망**해 시도 자체를 회피한다.
- 한 번 실패 경험이 생기면 **다시 도전하지 않는** 패턴이 고착된다.

### 2.2 목표 (MVP 기준)
1. 카페 키오스크 시뮬레이션 1종 완성
2. 튜토리얼 완주율 **70% 이상**
3. 자유 연습 → 실전 연습 전환율 **50% 이상**
4. "다시 해보고 싶다" 재시작 버튼 클릭률 측정 기반 재방문 의도 파악

## 3. 대상 사용자

### 3.1 주 타겟
| 항목 | 내용 |
|------|------|
| 연령 | 60–80세 |
| 디지털 경험 | 스마트폰 기본 사용 가능, 키오스크 경험 거의 없음 |
| 학습 동기 | "혼자서도 주문하고 싶다", "아이들한테 안 물어보고 싶다" |
| 사용 환경 | 태블릿 또는 스마트폰 세로 모드 |

### 3.2 보조 타겟
- 시니어 디지털 교육 강사 (수업 도구로 활용)
- 시니어 자녀·가족 (선물 목적으로 소개)

## 4. 서비스 범위 (MVP)

### 4.1 시뮬레이션 콘텐츠

| 콘텐츠 | 설명 | 우선순위 |
|--------|------|---------|
| 카페 키오스크 | 스타벅스·메가커피 스타일 무인 주문기 | P0 (완성) |
| 편의점 셀프계산대 | 바코드 스캔·현금·카드 결제 | P1 |
| 지하철 승차권 발매기 | 노선 선택·카드충전·1회권 구매 | P1 |
| 병원 무인접수기 | 예약 확인·접수·수납 | P2 |
| 은행 ATM | 잔액 조회·출금·이체 | P2 |

### 4.2 학습 모드 (카페 키오스크 기준)

| 모드 | 설명 | 힌트 |
|------|------|------|
| **튜토리얼** | 8단계 순서대로 안내, 다음 단계 자동 전환 | 말풍선 + 노란 하이라이트 강제 |
| **자유 연습** | 순서 제한 없이 자유롭게 탐색 | 도움말 오버레이 (선택) |
| **실전 연습** | 힌트 없이 실제 기기처럼 사용 | 없음 |

## 5. 화면 구성 (카페 키오스크)

```
홈 화면
 ├─ 튜토리얼 → 튜토리얼 안내 → 메뉴 → 메뉴 상세 → 장바구니 → 결제 → 완료
 ├─ 자유 연습 → 메뉴 → 메뉴 상세 → 장바구니 → 결제 → 완료
 └─ 실전 연습 → 메뉴 → 메뉴 상세 → 장바구니 → 결제 → 완료
```

### 5.1 화면별 핵심 요소

| 화면 | 주요 액션 | 핵심 UX 포인트 |
|------|-----------|--------------|
| 홈 | 모드 선택 (3개 버튼) | 모드 설명 한 줄씩, 선택 후 즉시 이동 |
| 튜토리얼 안내 | 시작하기 | 단계 수·소요 시간 미리 안내 |
| 메뉴 | 카테고리 탭 + 메뉴 카드 담기 | 커피/음료/디저트 18종, 이미지+이름+가격 |
| 메뉴 상세 | 옵션 선택 + 장바구니 담기 | 사이즈·온도·수량 선택 |
| 장바구니 | 수량 조절 + 결제하기 | 합계 금액 상시 노출 |
| 결제 | 결제 수단 선택 (카드/현금) | 카드 탭 → 삽입 애니메이션 → 승인 |
| 완료 | 다시 하기 / 홈으로 | 성취 메시지 + 영수증 요약 |

## 6. 기능 요구사항

### 6.1 튜토리얼 힌트 시스템
- 말풍선은 대상 요소 위/아래 자동 배치 (화면 밖 이탈 방지)
- 하이라이트 영역 외 화면은 반투명 오버레이로 비활성화
- 정해진 요소를 클릭해야만 다음 단계로 진행
- 잘못된 요소 클릭 시 "여기를 눌러보세요 👆" 말풍선 재표시

### 6.2 자유 연습 도움말
- 화면 우하단 고정 "도움말" 버튼 (48×48px 이상)
- 도움말 오버레이: 현재 화면에서 할 수 있는 행동 목록

### 6.3 장바구니
- 최대 10종 담기
- 동일 메뉴·동일 옵션 중복 시 수량 합산
- 수량 1 미만으로 감소 시 항목 자동 삭제 (삭제 확인 없이)

### 6.4 결제 시뮬레이션
- 카드 결제: 카드 삽입 애니메이션(2초) → "결제 완료" 전환
- 현금 결제: 금액 투입 UI → 거스름돈 안내
- 실제 결제 없음, 완전 시뮬레이션

### 6.5 진행 상태 저장
- `localStorage` 기반, 브라우저 새로고침 후에도 유지
- 저장 항목: 마지막 모드, 장바구니 내용, 튜토리얼 단계

## 8. 디자인 시스템

### 8.1 색상
| 토큰 | 값 | 용도 |
|------|----|------|
| `--color-primary` | `#122961` | 헤더, 주요 버튼, 강조 |
| `--color-secondary` | `#FCE300` | CTA 버튼, 힌트 하이라이트 |
| `--color-bg` | `#F5F5F5` | 페이지 배경 |
| `--color-surface` | `#FFFFFF` | 카드, 모달 배경 |
| `--color-success` | `#1A6B2A` | 완료·성공 상태 |

### 8.2 UX 원칙
1. **화면당 주요 행동 1–2개** — 선택지 과부하 금지
2. **부정적 단어 금지** — "오류" 대신 "다시 해볼까요?"
3. **모든 화면 좌상단에 뒤로가기** — 레이블 포함 필수
4. **행동 완료 후 명확한 피드백** — 담기·결제 완료 즉시 확인 메시지

## 9. 측정 지표 (KPI)

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| 튜토리얼 완주율 | ≥ 70% | 완료 화면 도달 수 / 튜토리얼 시작 수 |
| 자유 → 실전 전환율 | ≥ 50% | 실전 연습 시작 수 / 자유 연습 완료 수 |
| 재시작 클릭률 | 측정 후 목표 설정 | 완료 화면 "다시 하기" 클릭 수 |
| 힌트 요청률 | ≤ 30% | 도움말 버튼 클릭 수 / 자유 연습 세션 수 |
